A/B Testing

A/B testing is widely used in industry. It is part of what we commonly call "experiment Design".

Putting it simply, The objective of an experiment design is to figure out if the change of a feature would results in a change of another feature. To illustrate, A corporation may decide to test different pages designs. In medicine, research may want to test or experiment patent reactions to novel drugs

I - Experiment Design

To understand the process of experiment design, it is important to know some of the fundamental concept and the statistical techniques used in the process

1- Experiment concepts
Images that a fictional company name amazonic engineering team are discussing on a potential shopping pages changes. The discussion revolves around a specific feature that will display a different color blue as compared to an original yellow color. A experiment would help answer the following question. Does the change in color drives more sale? To helps illustrate how experiment design works, let's define the types of experiment that we are commonly used.

Based on the amount of control of the feature that are available, two keys aspect determine whether there is an experiment or not. Note that there are others that will not be explain in this article. Assuming we split the customers into two main groups, we must first have the possibility to establish a comparison between group A and group B. Second, we must be have control over other variables.. If the two concepts are not attainable, then we can not implement an experiment.

As noted above, if we have a higher control over the features, we can have an experiment. However, there are instances in which, there is no group or we are unable to complete a random sampling. In these types of scenarios, we can use alternate methods such as observational study. Observational study is used when we mainly don't have control over the features that we have.
eg. A medical team collected patients with type 1 diabetes data to compare the effect that the type of exercise completed after nutrition and glucose level (Observational study). Amazonic decided to send a coupons codes to some "primy" members (Experiment).
1-1 Other Types of Experiment
There are two types of experiment that we can implement, one that allow each group to only see oe version of the pages. If group A is asigned to interact with the yellow webpage mentioned above, they will not be able to use the webpage with blue color. The goes for group B. This is often called Between-subjects design.

The other method is called Within-subjects design and it allow both group to interact with the two designs. In this process, we can show group A the yellow page first then blue next and we reverse the other for group B. It is important to note that the customers are randomly selected.
1-2 Sampling methods
Random sample can be used and it gives each customers equal chance of being selected during the process. The sample should be representative of the population.
Another sampling techniques that can be used is stratified random sampling. This approach divides the population into group of interest such as geographical location and it randomly sample based on each region to ensure each group is representative of the population.
1-3 Measuring Outcomes
One should not confuse the goal of the experiment and the evaluation metrics. They are both similar but distinguishable because the goal could be different from the evaluation metrics. Some examples of video recommendation system metrics could be user, better performance if watch time is longer, better performance if more movie number of search, and better performance if higher video ranking.
1-4 Building metrics
Sticking wih our previous example of Amazonic group A and B, we can build the metrics by first creating metric funnel. A metric funnel is the steps that a use takes on the website. For instance, a user land on the home page, search a product by name, click on the lowest price filter, add to cart, and checkout. Using this principal, we can decide where to split our user called a Unit of Diversion A unit of diversion is the place that we can use to randomly split the group.

We can have different unit of diversion such as Event-based, Cookie-based, Account-based . In Event-based we can associate the user behavior with an event such as page view. Every time the user interacts the page, the experiment is triggered. With cookie-based, a cookie is assigned to the user device and tracks the experiment information. lastly, the account-based diversion requires the use of user information such id, phone number, or email to save the experiment details information.

Now that the unit of diversion is in place, we can cover the type of metrics. Based of the features, we can have an invariant metric and an evaluation metric. An invariant metric is a metric that remain consistent with between groups. It can be the number of cookies in each group. An evaluation metric focuses on each user during the experiment. One example of evaluation metric can be the amount of user of left the page without making a purchase.

1-5 Bias in experiment
In terms of bias, we can identify sampling bias, novelty bias, and experiment bias. Sampling bias occurs when our sampling process fail to collect a representative data from the population. novelty bias has to do with a user for instance behavior change because they are seeing something new. Experiment bias is where the experimenter might be unable to resist the temptation and push the participant toward taking steps that will generate the expected results.

1-6 Mnemonic
To conclude this section, one can use a mnemonic SMART to remember the necessary steps that needed to complete an experiment design.

Specific Having specific experiment goals.

Measurable Ensuring the metrics used are measurable

Achievable Ensuring the experiment approach is achievable

Relevant Ensuring the experiment relevant and has a purpose

Timely Ensuring the experiment has reasonable time limit of does not indefinitely
